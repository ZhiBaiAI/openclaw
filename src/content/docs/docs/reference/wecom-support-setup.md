如果你想把 OpenClaw 接进企业微信，先记住两件事：

- 当前官方支持的是**企业微信智能机器人**
- 想要机器人能主动给你发消息，创建时要选**长连接**

截至 2026 年 3 月 10 日，企业微信官方帮助页标题为《OpenClaw接入企业微信智能机器人》，这篇文档按那份官方说明整理成更适合站内查阅的版本。

## 前期准备

开始前，先确认你已经具备这些条件：

- 已安装最新版企业微信客户端
- OpenClaw 已经部署完成
- 你已经决定用哪种接入方式：
  - 腾讯云 Lighthouse 上的 OpenClaw
  - 本地终端部署的 OpenClaw

## 第一步：创建企业微信智能机器人

在企业微信客户端里按下面的路径创建机器人：

1. 进入工作台
2. 打开智能机器人
3. 选择创建机器人
4. 选择 API 模式
5. 选择**长连接**

创建后记下这两个值，后面配置会用到：

- `Bot ID`
- `Secret`

如果这里没选长连接，后面就不能按官方这套方式完成主动消息和配对流程。

## 方式一：腾讯云 Lighthouse 部署的 OpenClaw

如果你的 OpenClaw 跑在腾讯云 Lighthouse 上，官方推荐在实例管理界面里直接完成关联：

1. 打开腾讯云 Lighthouse 控制台
2. 进入已部署 OpenClaw 的轻量应用服务器实例
3. 进入“应用管理”
4. 在通道里选择**企微机器人（长连接）**
5. 填入前面创建机器人时拿到的 `Bot ID` 和 `Secret`
6. 点击“添加并应用”
7. 按提示完成重启
8. 回到企业微信机器人创建页，保存并创建

完成后，你就可以直接在企业微信里和 OpenClaw 对话。

## 方式二：本地终端部署的 OpenClaw

如果你是在本地设备、普通云服务器或其他非 Lighthouse 环境里部署 OpenClaw，按官方帮助页整理后的流程如下。

### 1. 安装企业微信插件

```bash
openclaw plugins install @wecom/wecom-openclaw-plugin
```

### 2. 启动或重启 OpenClaw 网关

```bash
openclaw gateway start
```

如果你的网关已经在运行，也可以先确认状态，再决定是否重启。

### 3. 添加企业微信渠道

```bash
openclaw channels add
```

在交互式配置里，按下面的思路填写：

- `select channel`：选择**企业微信**
- `Bot ID`：填你在企业微信里创建机器人后拿到的值
- `Secret`：填对应密钥
- 结束时选择 `finish`

### 4. 完成配对

渠道添加完成后，继续按界面提示完成配对：

1. 选择配对方式 `Pairing`
2. 保存机器人，并在企业微信里给它发一条消息
3. 企业微信会返回一段配对信息
4. 复制这段信息的**最后一行**
5. 粘贴回终端，完成配对

配对成功后，企业微信里就可以正常和 OpenClaw 对话。

## 成功接通后的判断标准

满足下面几条，基本就说明企业微信渠道已经可用：

- 机器人能在企业微信里收到你的消息
- OpenClaw 能正常回消息
- 重启服务后仍然可以继续对话
- 如果涉及配对，配对提示已经完成，没有卡在待确认状态

## 常见卡点

### 1. 机器人创建时没选长连接

这是最常见的问题之一。官方这篇接入说明明确是基于**长连接机器人**写的。

### 2. Bot ID 或 Secret 填错

建议直接复制粘贴，不要手敲。

### 3. 插件没安装就开始加渠道

本地终端部署时，先装插件，再执行 `openclaw channels add`。

### 4. 配对信息没复制最后一行

官方流程里明确要求把企业微信返回信息的最后一行贴回终端，否则配对可能会卡住。

## 相关延伸

企业微信官方这篇说明还提到了两个后续方向：

- 智能表格 Webhook：适合把外部数据写入智能表格
- 企业微信 API：适合继续调用企业文档等能力

如果你当前目标只是先把聊天通道接通，可以先把机器人对话跑通，再考虑这些扩展能力。
