## 安装脚本跑不动怎么办

先看两件事：

- 当前网络能不能正常访问安装脚本地址
- Node.js 版本是否过低

原站提到 OpenClaw 需要 Node.js 22+。如果本机版本明显太旧，先升级再继续。

## 装好了，但聊天工具没有回复

优先排查下面几项：

- Bot Token 或接入参数是否填错
- 你的用户 ID 是否真的被加入允许列表
- 是否还有待批准的配对请求
- 模型接入方式是否有效
- 后台日志里有没有明显报错

如果你是 Telegram 方案，记得检查有没有待批准的 pairing。

## 一开始能回复，过一会儿又不回复了

这类情况最常见的原因，不是模型坏了，而是后台没有继续常驻。

优先检查：

- 设备是不是休眠、关机或断网了
- 后台服务是不是退出了
- 如果是本地电脑方案，你是不是把机器合盖了

OpenClaw 需要后台持续运行。

如果你更希望它长期在线，云服务器会比本地电脑更稳。

## 想重新走配置流程怎么办

通常有两种做法：

- 重新进入完整向导：`openclaw onboard`
- 只修改部分配置：`openclaw configure`

## 授权后生成的 token 文件能不能乱放

不能。

无论是邮箱、日历还是其他 OAuth 授权，生成的 token 文件都相当于钥匙。

建议：

- 不要提交到 Git
- 不要发到公开场合
- 给文件设置合理权限

## 为什么我觉得它“不像我的助手”

大多数时候不是模型不行，而是规则还不够清楚。

优先去补：

- SOUL.md
- USER.md
- AGENTS.md

尤其是风格、边界、主动提醒频率这几块，通常要在实际使用一段时间后再调一次才会顺。

## 自动提醒太多，感觉被打扰怎么办

优先改这些地方：

- 拉长心跳间隔
- 删除价值不高的检查项
- 把普通事项改成汇总消息
- 深夜和周末减少主动提醒

主动是加分项，打扰是减分项。
