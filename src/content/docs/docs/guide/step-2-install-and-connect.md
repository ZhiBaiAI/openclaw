这一步的目标很明确：

安装完成后，你能在自己的聊天工具里对 OpenClaw 说第一句话，并收到它的回复。

## 先选运行位置

OpenClaw 需要一个“常驻的地方”来运行。

常见有两种做法：

### 方案 A：放在云服务器

适合：

- 想让它 24 小时在线
- 不想长期占用自己的电脑
- 想随时从手机上找它

一般入门配置就够用：

- Ubuntu 22.04
- 2 核 CPU
- 4GB 内存
- 40GB SSD

这类配置通常已经足够开始使用。

### 方案 B：放在自己的电脑或旧设备

适合：

- 先低成本试用
- 不介意设备关机后助手暂时离线
- 想把数据尽量留在家里的设备里

家里闲置的 Mac mini、旧笔记本、小主机，都可以作为入门机器。

## 安装前你至少要准备好

- 一台能长期运行的设备
- 稳定网络
- 一个可用的 AI 模型接入方式
- 一个你打算拿来聊天的渠道，最常见是 Telegram，也可以接企业微信

## Windows 用户先做这一步

如果你用的是 Windows，建议先安装 WSL2，再在 WSL 里的 Linux 环境中操作。

这样做的好处是：

- 跟官方推荐的 Linux 运行环境更接近
- 安装脚本和命令行体验更稳定
- 后续排错、看日志、装依赖都会省事很多

安装方式很简单：以管理员身份打开 PowerShell，执行下面这条命令即可：

```powershell
wsl --install
```

安装完成后，按提示重启电脑，再进入 WSL 继续后面的安装步骤。

## 一键安装

原站给出的安装方式是执行官方安装脚本，自动补依赖并进入向导：

`curl -fsSL https://openclaw.ai/install.sh | bash`

实际体验上，这一步的好处是你不用先手动装一堆依赖，再一个个配置。

## 向导里最关键的几步

安装脚本跑完以后，会进入交互式配置。第一次配置时，优先关注这几项：

### 1. 先选简单模式

如果你是第一次装，优先选快速模式，不要一开始就钻高级选项。

目标只有一个：

先跑起来。

### 2. 选模型接入方式

你需要给 OpenClaw 一个可用的“大脑”。

比较常见的做法是：

- 用 Claude 订阅对应的接入方式
- 或者使用 Anthropic API Key
- 如果你想用百炼平台里的模型，可以直接看 [百炼平台模型配置示例](../reference/bailian-model-config-example/)
- 如果你有其他兼容模型，也可以后面再换

刚开始不需要为了模型选择纠结太久，先选一个稳定可用的方案。

### 3. 连接聊天渠道

最常见的入门方式是接 Telegram。

如果你想接企业微信，可以直接看 [企业微信支持配置](../reference/wecom-support-setup/)。

原因很简单：

- 手机上好用
- 随时能找助手
- 适合后续做提醒和主动消息

如果你用 Telegram，记得提前拿到自己的 Telegram 用户 ID，并确保只有你自己的账号被加入管理员或可对话名单。

## 首次成功的判断标准

满足下面三条，这一步就算过关：

- 你能在聊天工具里找到自己的助手
- 你给它发消息，它会回复
- 你能看懂怎么重新打开配置或看日志

## 别忘了它需要后台常驻

很多人第一次装好以后，会忽略一件很关键的事：

OpenClaw 不是点开网页用一下就结束的工具，它需要后台服务一直运行。

这意味着：

- 云服务器方案更适合长期在线
- 本地电脑方案只要关机、休眠或断网，助手就会暂时离线
- 聊天工具里“不回复”时，先别急着怀疑模型，先确认后台服务还活着

如果你是先用本地机器试玩，这是正常现象，不是配置一定有问题。

## 第一次接通后，建议立刻做 3 个测试

1. 发一条最简单的消息，比如“你好”，确认它能正常回复
2. 如果你用的是 Telegram 或企业微信，确认有没有待批准的 pairing 或配对请求
3. 重启一次后台服务，再发一条消息，确认不是“只成功一次”

## 安装后最常用的几条命令

这几条建议先记住：

- `openclaw status`
- `openclaw health`
- `openclaw configure`
- `openclaw daemon restart`
- `openclaw daemon logs`

先记住这几条，已经足够处理大部分入门阶段的问题。

## 先不要追求“功能全”

很多人安装成功后，会立刻想把邮箱、日历、浏览器、技能包、自动化一次全接上。

不建议这样做。

更稳妥的节奏是：

1. 先保证能正常聊天
2. 下一步再去调风格和边界
3. 后续再逐步接外部工具

这样出问题也更容易定位。
