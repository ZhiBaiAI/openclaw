这套文档更适合两类人：

- 已经完成交付，想继续把 OpenClaw 用顺手的人
- 准备自己部署，想按一条稳定路线往下走的人

## 如果你已经完成交付

建议按下面这个顺序用：

1. 先看 [第 2 步：把助手装起来并连上聊天工具](./guide/step-2-install-and-connect/)，确认部署和入口都正常。
2. 再看 [第 3 步：让它更像“你的助手”](./guide/step-3-make-it-yours/)，把规则和边界补齐。
3. 需要排错时直接看 [常用命令速查](./reference/common-commands/) 和 [常见问题](./faq/common-issues/)。
4. 想继续扩展时，再往后看第 4 步到第 7 步。

## 如果你还没交付，只是先来看看

建议先回到站点首页和这些页面：

- [方案能力](../features/)：先判断方案适不适合你的场景
- [真实案例](../cases/)：先看真实案例是不是你需要的方向
- [交付服务](../service/)：再看交付范围和支持方式

## 时间不多，先看这几篇

- [第 1 步：先明白 OpenClaw 能帮你做什么](./guide/step-1-what-openclaw-can-do/)
- [第 2 步：把助手装起来并连上聊天工具](./guide/step-2-install-and-connect/)
- [常用命令速查](./reference/common-commands/)
- [常见问题](./faq/common-issues/)

## 这套文档能解决什么

- 让你知道交付后应该先看哪一部分，而不是一上来就翻全部命令
- 把安装、接入、规则、自动化和安全分开讲清楚
- 出现问题时能快速定位到命令、FAQ 和支持资料
- 后续扩展技能、工具和自动化时有明确入口
